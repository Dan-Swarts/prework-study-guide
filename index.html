HTML


HTML defines the structure of a web page


HTML is made out of "elements"




Main elements: these are present in every HTML page, and are the most basic/most important

  • The <html> element is the root element: the stating point for every HTML page

  • The <head> element contains information about the web page that the user doesn't see

  • The <body> element contains all content that the user will see on the page

Content elements: These appear in the body, and show all the content visible to the user

  • The <h1>,<h2>,<h3>,<h4>,<h5>, and <h6> tags stand for "header 1" through "header 6." Header 1 is biggest, and 6 is smallest

  • The <p> tag stands for "paragraph", and represents some text

  • The <img> tag stands for "image," Which displays a picture on the page

  • The <ul> tag stands for "unordered list," which is a bulleted list

  • The <ol> tag stands for "ordered list," which is a numbered list

  • The <li> tag stands for "list item," and is one entry in the list

  • The <a> tag stands for "anchor," wrap this around another element to create a hyperlink

Semantic elements: these aren't content per say, but are dividers which hold other elements. They are important for defining structure of the page, making it more readable

  • <section> one section or division of the web page

  • <footer> final content

  • <header> introductory or navigational content

  • <figure> illustrations, diagrams, photos, etc.

  • <figcaption> the caption for a figure

  • <article> independent, self-contained content

  • <aside> something indirectly related to surrounding content

  • <details> gives additional details which can be hidden by the user

  • <summary> acts as a heading for <details>

  • <main> the main content of the document

  • <mark> marked/highlighted text

  • <nav> gives navigational links

  • <time> date/time

  • <div> and <span> these are non-semantic elements, they don't describe the content contained. However, it still acts as container for content.

CSS


CSS is a way to orgainze the attributes of HTML elements


CSS is made up of "rule sets"



The selector defines which elements are being targeted by the declaration


The declaration gives the attribues which will apply to the selected elements


Together, the selector and declaration apply the attribues which define how a web page looks

Selectors: how to target the right elements




Declarations: attribues you can apply to the elements

  • These exist in key-value pairs: the attribute to modify, and its value

  • The value can be expressed in pixels, %size of the page, or other values

  • This can be used to change the color of the element, the height, width, etc.

Git

Git is a version control system, which allows multiple people to work on a codebase at the same time

Set up

  • Dowload Git Bash, the shell which allows you to run Git commands

  • Use commands through the Git Bash terminal

  • Using Git Bash, type `git config --global user.name "<name>"` to set the name used for all git pushes on the current user

  • type `git config --global user.email "<email>"` to set the email for all git pushes for this user

  • These commands are important because it lets Git track who contributes code to the project

  • use git config `--global color.ui auto` to set automatic command line coloring

GitHub

  • GitHub is a platform for hosting and sharing projects managed with Git version control

  • You will need to set up a GitHub account and find the project you're contribuing to, and get the ssh url

  • Navigate to the directory where you want to work on the project, and clone the project using `git cone [url]`

Features of GitHub

  • Git stores every version of the project, including all different changes in previous versions

  • GitHub lets you create "issues" which can let you manage your different tasks, and focus your efforts

  • You can create your own "branch," or an isolated copy of the project where you can address issues without intrducing bugs to the stable version

  • recieve changes made to the repo by using `git pull origin <branch name>

Branching

  • once you're in your workspace, and you've cloned the directory, use the `git branch` and `git checkout` commands to manage your branching

  • generate a new branch using `git checkout -b <branch name>`

  • move in between existing branches using `git checkout <branch name>`

  • see all existing branches using `git branch`

  • once your code is completed and tested in your own branch, use `git add -A` to add changes, `git commit -m <message> to stage changes with a comment on what's been changed, and finally `git push origin <branch name> to put the changes in the cloud
  • Finally, have some other developers review your code and then merge the branch into the stable version
